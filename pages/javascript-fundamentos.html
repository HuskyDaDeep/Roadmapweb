Bloco 3: Fundamentos do Javascript

Apos ter visto como o HTML e CSS se comportam, vamos olhar uma linguagem de programação!

O que é Javascript

JavaScript (JS) é uma linguagem de programação amplamente utilizada para tornar as páginas da web dinâmicas e interativas. Originalmente, ela foi criada para rodar nos navegadores (lado do cliente), permitindo coisas como:

  • Conteúdo dinâmico: Atualizar partes de uma página sem precisar recarregá-la completamente.
  • Interatividade: Criar menus suspensos, formulários que validam informações em tempo real, galerias de imagens, animações e muito mais, tudo em resposta a ações do usuário (cliques, movimentos do mouse, etc.).
  • Controle de multimídia: Tocar e pausar vídeos, controlar o volume de áudio, etc.

O conceito central aqui é o **algoritmo**: uma sequência finita de instruções bem definidas e não ambíguas, usada para resolver um problema ou executar uma tarefa.

Pensando em Algoritmos: Conceitos Chave

  • Variáveis: Armazenando Informações

    Imagine uma **variável** como uma caixa rotulada onde você pode guardar diferentes tipos de informações. O rótulo é o nome da variável, e o que está dentro da caixa é o seu valor.

    
// Exemplo em um pseudocódigo
NOME = "João Silva"
IDADE = 30
TEM_CARRO = VERDADEIRO

  • Tipos de Dados: A Natureza da Informação

    Os dados que guardamos nas variáveis podem ter diferentes "tipos", que definem o que podemos fazer com eles:

    • **Texto (String):** Sequências de caracteres (ex: "Olá mundo", "Meu nome é...").
    • **Número (Number/Integer/Float):** Valores numéricos (ex: 10, 3.14, -5).
    • **Booleano (Boolean):** Representa "verdadeiro" ou "falso" (ex: `true`, `false`).

  • Operadores: Realizando Ações

    São símbolos que usamos para realizar operações com os dados:

    • **Aritméticos:** `+` (soma), `-` (subtração), `*` (multiplicação), `/` (divisão).
    • **Comparação:** `==` (igual a), `!=` (diferente de), `>` (maior que), `<` (menor que), `>=` (maior ou igual), `<=` (menor ou igual).
    • **Lógicos:** `E` (AND - todas as condições verdadeiras), `OU` (OR - pelo menos uma condição verdadeira), `NÃO` (NOT - inverte o valor booleano).

  • Estruturas Condicionais (Se/Então/Senão): Tomando Decisões

    Permitem que seu algoritmo tome diferentes caminhos com base em uma condição. "Se algo for verdadeiro, faça isso; senão, faça aquilo."

    
// Exemplo em pseudocódigo
SE (temperatura > 25) ENTÃO
    ESCREVER "Ligar ar-condicionado"
SENÃO
    ESCREVER "Manter ar-condicionado desligado"
FIM_SE

  • Estruturas de Repetição (Laços): Repetindo Tarefas

    Permitem que um conjunto de instruções seja executado várias vezes, até que uma condição seja satisfeita.

    
// Exemplo em pseudocódigo (CONTADOR de 1 até 5)
PARA CONTADOR DE 1 ATÉ 5 FAÇA
    ESCREVER CONTADOR
FIM_PARA

// Saída: 1 2 3 4 5
